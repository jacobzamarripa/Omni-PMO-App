Dailies Command Center — Help
Reference guide for reviewers and managers
Daily Workflow
1
Sync QB — Pull the latest project data from QuickBase into the reference sheet. Do this when QB statuses or BOMs have changed.
2
Run Review — Re-run the diagnostic engine for a specific date. Prompts for a date, processes daily files, and reloads the queue.
3
Refresh — Fast reload of the current mirror sheet. Use when corrections have been made directly in the sheet.
4
Select a project — Click any item in the Diag. Queue. The detail panel opens. On mobile, tap Prev / Next to move through projects without committing or skipping.
5
Review — Read the Vendor Update, check the System Diagnostic flags, and review the project schedule and velocity cards.
6
Write your note — The OMNI Manager Note pre-fills with an AI-drafted summary. Edit it to reflect your actual decision. Your edits are tracked separately from the auto-draft.
7
Commit or Skip — Commit saves the review and moves the project to the Reviewed drawer. Skip (Clear) removes it from the queue without saving.
8
Export All — Click Export All in the Reviewed drawer to email the daily summary CSV.
Navigation Bar
⟳ Sync QB
Pulls fresh project data from QuickBase API directly into the reference sheet. Required before running a review if QB data has changed.
▶ Run Review
Prompts for a date and re-runs the full diagnostic engine — processes daily files and reloads the queue.
↻ Refresh
Fast reload from the existing mirror sheet without re-running diagnostics. Use after making manual corrections in the sheet.
AI badge
Shows how many Gemini API calls have been used today out of the daily limit. Green = OK, yellow = nearing limit, red = at limit.
Projects
Total active projects in the current daily run.
BSLs
Total build-site locations (homes passed) across all active projects.
Critical
Projects with at least one critical flag. Click to toggle Critical-only filter on/off.
Ref Data age
Shows the age of the QB reference data. FRESH = synced today. STALE = data older than expected — run Sync QB.
☀ / 🌙
Toggle dark/light mode. The app also follows your system preference automatically.
Reviewed tab
The Reviewed drawer tab is always visible on the right edge of the screen. Click it to slide the drawer open and see committed projects or export the daily summary.
Gantt resize
Drag the thin bar above the Gantt chart up or down to resize its height.
Mobile Navigation
◀ Back
Return to the project queue from the detail view.
‹ Prev / Next ›
Move to the previous or next project in the queue without committing or skipping. Disabled at the first or last item.
Reviewed
Opens the Reviewed bottom sheet showing all committed projects for the session.
Skip (Clear)
Removes the current project from the queue without saving a review.
✔ Commit
Saves your manager note and moves the project to the Reviewed list. The next project opens automatically.
Diagnostic Queue
Check Crossings
Count of projects flagged for a special road or utility crossing. Click to filter the queue to only those projects.
QB Status
Count of projects with a QuickBase status mismatch or missing status. Click to filter. This is an admin action — the status in QB needs to be updated to match field reality.
Filters
Reveals the filter strip to narrow the queue by severity, vendor, city, or OFS forecast date. The button turns red when any filter is active.
☰ / ⊞
Toggle between card view (default) and compact list view for the queue.
Project Schedule Card
Shows the construction timeline for the selected project.
CX Start / Complete
The official construction start and completion dates from QuickBase.
Time Bar
Visual progress from CX Start to CX Complete. The bead shows today's position. Stars mark 100% completion milestones for each phase (UG, AE, Fiber, NAP).
Target Date
The projected OFS (On For Service) date. Turns red when overdue.
X-ING
A special crossing is present on this project (road bore, railroad, waterway, etc.). Extra permits and restoration scope may apply — verify with the vendor.
Field Velocity & Production
Real-time production progress pulled from the daily report.
UG / Fiber / Splicing / NAP
Each bar shows completed vs. BOM quantity and the daily pace needed to finish on time.
Days Left
Calendar days remaining until the target date. Zero or negative means the project is at or past its deadline.
BSLs
Total homes in scope for this project.
Pending Light
Light has not yet been confirmed at cabinets. No fiber or splicing can begin until light is present.
Diagnostic & Manager Review
Vendor Update
The most recent comment submitted by the vendor in their daily report. This is unedited — it reflects exactly what the crew reported.
System Diagnostics
Auto-detected flags generated by the engine. Each flag represents a data discrepancy or condition that may require action. See the Flag Glossary below.
OMNI Manager Note
Your field for the daily decision or action taken on this project. The AI pre-drafts a note based on the flags — shown in muted italic text. Edit it before committing. Once you type, the draft is replaced with your words.
CD Intelligence
CD Intel
Only appears when a construction drawing has been analyzed by AI (Gemini). Summarizes complexity factors from the CD — bore depths, crossing counts, restoration scope, special conditions. Use this to verify whether the vendor's reported scope matches the design intent.
Global Construction Timeline (Gantt)
Shows all active projects as bars across a shared calendar. Automatically sizes to fit the number of projects.
Bar Color
Matches the project's current stage/status. Overdue projects display in red. Hover a bar to see a summary HUD.
Beads
Small markers on the bar: L = Locates called, C = Cabinets set, ⚡ = Light present, ★ = Phase 100% complete.
EOM Spike
A concentration of projects ending at or near end-of-month. Hover for a breakdown of which vendors and BSL counts are involved.
Theater Mode
Click any project row in the Gantt to highlight it and dim others. Click again or press Escape to exit. Use Fullscreen to expand the Gantt edge-to-edge.
Flag Glossary
ID Auto-Corrected
The vendor submitted under a slightly different FDH ID. The engine matched it to the correct project automatically — confirm the vendor is using the right ID going forward.
Status Mismatch
The project's QB status does not match what the field report indicates. Admin action required: update the status in QuickBase.
Missing QB Status
No status was found in QB for this project. The record may be missing or incorrectly named. Admin action required.
Check Crossings
One or more special crossings detected on this project. Confirm permit status, bore depths, and restoration plan with the vendor.
BOM Discrepancy
Reported quantities exceed the BOM (Bill of Materials). Either the scope has expanded or the vendor over-reported. Verify with the PM.
Scope Deviation (NAP)
NAP count reported by the vendor does not match the BOM. Could indicate a scope change, a partial installation, or a reporting error.
Possible Reroute
Field conditions suggest the route may have changed from the original design. A new CD or field sketch may be needed.
Lighting Risk
Light-to-cabinets is not confirmed and the project is approaching or past the point where fiber work should have started. Escalate if blocking production.
Pending Light
Light has been called for but not yet confirmed at the cabinet. No fiber or splice work can begin.
Refresh Ref Data
An admin action cleared a mismatch flag — the reference data in the system has been updated and a fresh sync is recommended to reflect the correction.
