Omni PMO App — Help
Reference guide for reviewers and managers
Workspaces & Views
Detail View
The standard QA view. Focuses on one project at a time, providing full schedule, velocity, and AI diagnostic breakdowns.
Grid View
High-density overview. Displays multiple projects at once. Use the pivot selector in the queue controls to switch between Schedule and Velocity data.
Inbox View
Email-style queue lane with full chips and wider row rhythm for scanning project health.
List View
Denser queue mode for quick FDH triage when you want tighter rows and less tag noise.
Daily Workflow
1
Sync QB pulls the latest project data from QuickBase.
2
Run Review processes daily vendor files and loads anomalies into the queue.
3
Review Hub is where crossings, status mismatches, and BOM checks get cleared.
4
Commit saves a PM review and moves that project forward.
Flag Glossary
Status Mismatch
Project status in QuickBase does not match the field report and needs review.
Check Crossings
Special crossings were detected and need permit or construction verification.
Please Input BOM
Main BOM quantities are zero, which blocks velocity calculations.
Ghost Report
A project is marked active but the vendor did not submit a daily report.
CD Intel
AI detected noteworthy crossing or design risk in the construction drawing set.
