# Domain Context: Omni PMO App

## Glossary
### Projects & Identification
- **FDH ID**: The canonical primary identifier for a project (e.g., `ANN01-F04`). Also referred to in code/sheets as `FDH Engineering ID`.
- **Active Portfolio**: The set of projects currently under management and visible in the dashboard. Eligibility is determined by stage, status, and reporting grace windows.
- **Inferred**: A value (Stage or Status) calculated by the engine based on observed production behavior (e.g., "Inferred CX" due to footage reports).
- **Target**: A scheduled or contractual value, typically sourced from Quickbase or a baseline (e.g., `Target Date`, `Target Completion Date`).
- **Stale**: A project state where no daily production reports have been received within the expected reporting window (usually 24-48 hours).
- **Ghost**: A project that exists in the **Active Portfolio** (according to Quickbase or Reference Data) but has no corresponding production records in the **Master Archive**.

### Reconciliation & Auditing
- **Reconciliation**: The process of aligning external systems (Quickbase) with the internal system of record (Master Archive). Example: `Work Log Reconciliation`.
- **Adjustment**: A numerical correction record (e.g., `LIFETIME RECONCILE`) created during reconciliation to align totals.
- **Audit**: Internal verification of the Master Archive against backups or historical logs to ensure structural integrity.
- **Heal**: A structural repair to the database schema or column alignment (e.g., fixing shifted columns).
- **Repair**: A data-cleaning task that corrects individual cell values (e.g., correcting an FDH ID typo in historical rows).
- **Soft Match**: An automatic resolution of a typo or identifier mismatch by the engine during processing (e.g., "Archive has typo. Matched to BCT01-F43").
- **Discrepancy**: A numerical mismatch in production metrics (e.g., footage or NAP counts) between systems or between daily reports and running totals.
- **Mismatch**: A structural or identifier conflict, such as an FDH ID or Vendor name that exists in one system but is missing or differently named in another.

### Findings & Alerts
- **Special Crossing**: A physical attribute or barrier of a project (e.g., Highway, RR, River) that requires specific permitting or verification.
- **CD Analysis**: (Construction Drawing Analysis) The process, either automated (Gemini Vision) or manual, of reviewing project PDFs to identify engineering details like crossings.
- **External Finding**: A data container used to stage information discovered outside of standard daily reports (e.g., a crossing found in a CD) before it is committed to Quickbase.
- **Staged Finding**: An edit (e.g., a new Manager Note or Crossing status) that has been entered in the Action Center but not yet committed to Quickbase.
- **BOM Comparison**: The reconciliation task of aligning Quickbase bill-of-materials quantities with actual vendor-reported usage.
- **Signal Alert**: A notification generated by the **Signal** system when a change is detected in tracked Drive folders or Quickbase tables.

### Communication & Feedback
- **Manager Note**: Authoritative communication entered by an Omni PMO manager, tracked in Quickbase (FID 188) and intended for external stakeholders.
- **Vendor Comment**: Raw feedback received from the field via daily production reports.
- **Gemini Insight**: Diagnostic or summarizing text generated by AI to assist in decision-making and narrative drafting.

### Workspaces & Views
- **Action Center**: The high-level workspace for **Reconciliation** (Master Archive vs. Quickbase), bulk status updates, and resolving **Special Crossing** and **BOM Comparison** discrepancies.
- **Diagnostic Hub**: (Formerly Critical Hub) The categorized view of operational risks, split into buckets like `Lighting / Flow Risk` and `Date / Timing Issue`.
- **Review Hub**: The administrative interface for reviewing daily production logs and committing them to the Master Archive.
- **Signal**: The change-detection and notification system that monitors Quickbase table updates and Drive folder activity.

### Milestones & Metrics
- **OFS Date**: (Original Forecasted Start) The canonical baseline start date. Formally identified as `OFS DATE` in reference data.
- **CX Start / CX Complete**: Construction Start and Construction Complete dates.
- **Production Metrics**:
    - **UG**: Underground footage.
    - **ST**: Strand footage.
    - **FB**: Fiber footage.
    - **NAP / Encl**: Network Access Points / Enclosures completed.

### Systems & Storage
- **Quickbase (QB)**: The external enterprise system of record.
- **Master Archive**: The internal Google Sheets database (`2-Master_Archive`) containing the full history of production logs.
- **Daily Review**: The "Mirror" sheet (`3-Daily_Review`) used by the engine to analyze current project health.
